Управление экспертизой
Модуль «Управление экспертизой» реализует функционал для проведения независимыми экспертами технической оценки заявок на гранты. Он охватывает полный цикл экспертной работы: от получения списка назначенных заявок до отправки заключения и уведомления участников процесса.

Участники
Основным пользователем модуля является независимый эксперт. Система также взаимодействует со следующими компонентами:
— Служба авторизации проверяет права доступа эксперта;
— Система уведомления оповещает соискателей об изменении статуса заявки;
— Внешняя система экспертизы (через API) фиксирует итоговое решение по заявке в общей экосистеме грантов.

Основные сценарии
Авторизация эксперта
Перед началом работы система подтверждает, что пользователь имеет роль «Эксперт» и действующую сессию.
Получение списка назначенных заявок
Эксперт запрашивает список заявок, назначенных ему для оценки. Система возвращает только те заявки, которые находятся в статусе «назначена» и привязаны к его идентификатору.
Просмотр заявки и приложенных документов
После выбора заявки система загружает её метаданные и полный пакет документов для ознакомления.
Отправка результата экспертизы
Эксперт заполняет структурированное заключение, включающее вердикт, количественную оценку и текстовые комментарии. Система:
— сохраняет результат в базе данных,
— передаёт итоговое решение во внешнюю систему экспертизы через адаптер,
— обновляет статус заявки на «Экспертиза завершена».
Уведомление соискателя
Как только экспертиза завершена, соискатель получает уведомление о новом статусе своей заявки.
Архитектурные компоненты
Модуль построен по принципу фасада. Взаимодействие с пользовательским интерфейсом осуществляется через ExpertiseManagementFacade, который координирует работу следующих компонентов:

IExpertiseRepository — отвечает за хранение и выборку назначений экспертиз и экспертных заключений.
IGrantApplicationRepository — предоставляет данные заявок, включая приложенные документы.
AuthorizationService — проверяет, имеет ли эксперт право работать с конкретной заявкой.
ExpertiseAdapter — преобразует внутренние данные в формат, принимаемый внешним API системы экспертизы.
NotificationService — отправляет уведомления соискателям через выбранный канал связи.
Взаимодействие с другими модулями
Модуль тесно связан с другими частями системы:
— Получает данные о заявках из модуля «Управление заявкой».
— Предоставляет информацию о результатах экспертиз модулю «Статистика грантов» для формирования аналитических отчётов.
— Использует общие сервисы: службу авторизации, систему уведомления и внешнюю систему экспертизы.

Безопасность
Безопасность обеспечивается на каждом этапе работы:
— Любая операция с заявкой требует предварительной проверки роли пользователя как «Expert».
— Эксперт имеет доступ только к тем заявкам, которые были ему явно назначены системой.
